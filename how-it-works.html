Splink

How it Works

Take this HTML document as an example: 


            <head>
            	<!-- a bunch of stuff in the head -->
            </head>
             
            <body>
            	<h1>Blog</h1>
             
            	<article class="blog-post">
            		<h2>Blog Post</h2>
            		<p>
            			<!-- the content of the blog post -->
            		</p>
            	</article>
             
            	<nav class="post-navigation">
            		<a class="splink-link" href="/posts/this-is-the-post-name">Previous Post</a>
            		<a class="splink-link" href="/posts/another-post-name">Next Post</a>
            	</nav>
             
            	<footer>
            		<p>
            			Find me on
            			<a href="http://github.com">Github</a> or 
            			<a href="http://twitter.com">Twitter</a>
            		</p>
            		<p>
            			© 2013 - Blog Author
            		</p>
            	</footer>
             
            </body>

The rest of your blog posts are structured the same way, namely with the content in an element that matches the CSS selector article.blog-post, and the navigation in nav.post-navigation. The footer stays the same, as does the <h1&rt; at the top. So we're targeting article.blog-post and nav.post-navigation. The links that take us from page to page are in the nav element and have the class .splink-link. So the plugin usage in this case would be:


            $("a.splink-link").splink(
              [{target: "article.blog-post", selector: "same"},
              {target: "nav.post-navigation", selector: "same"}],
              {
              	prefetch: true, 
              	animate: 250,
              	runScripts: true
              }
            );
